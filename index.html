摘要

本文介绍了一个面向多行业的检索增强生成（RAG）系统，旨在解决大语言模型在专业领域知识不足的问题。 系统采用先进的向量检索技术，结合语义理解能力，为用户提供准确、可靠的专业领域问答服务。

实测表明，该系统在多个行业基准测试中表现优异，检索准确率达到 90%+， 回答质量显著优于传统检索方法，为专业领域的知识获取提供了可靠的技术解决方案。

核心特性

🔍
多行业知识库
覆盖 12+ 个垂直行业，每个行业拥有独立的知识库和向量索引，支持专业领域的精准检索。
⚡
高效向量检索
基于向量相似度和关键词混合加权搜索，支持百万级文档的秒级检索，结合重排序提升结果相关性。
🧠
智能问答生成
利用大语言模型理解复杂问题，结合检索到的上下文生成准确、连贯的专业回答。
📊
多模态支持
支持文本、表格、图片等多种文档类型的解析和检索，充分利用文档中的多模态信息。
🔐
用户认证系统
完善的用户注册、登录和权限管理，支持对话历史记录和个性化设置。
现代化界面
响应式设计，支持明暗主题切换，提供流畅的用户交互体验和实时的检索进度展示。

技术细节

文档处理流程

系统采用自动化的文档处理流水线，支持 Excel、PDF、Word 等多种格式的文档导入。 文档经过文本提取、分块（Chunking）、向量化（Embedding）后存储成多层关联向量数据库。

# 文档处理流水线
docs.jsonl → 文本提取 → 分块 → Embedding → FAISS索引

# 检索流程
用户Query → Query Embedding → FAISS检索 → 重排序 → 上下文构建 → LLM生成

检索评估指标

指标 描述 目标值
Recall@K 前K个结果中相关文档的比例 ≥ 90%
MRR 平均倒数排名 ≥ 0.8
NDCG@K 归一化折损累积增益 ≥ 0.85
响应时间 端到端检索+生成时间 < 5s

快速开始

环境要求

Python >= 3.8
pip install -r requirements.txt

启动服务

# 启动 Web 服务
cd main
python app.py

# 访问 http://localhost:5002

构建知识库

# 从 Excel 文件构建 RAG 知识库
python scripts/build_rag_from_excel.py --input data.xlsx --output ./行业_RAG

# 批量处理流水线
python scripts/run_batch_pipeline.py
